TV Script Generation using RNN

Train a Recurrent Neural Network to generate a fake TV/film script.

TV Script Generation using RNN

November 10, 2021

⚡Text Generation
⚡Recurrent Neural Network (RNN)
⚡Gated Recurrent Unit (GRU)
⚡PyTorch

Text Generation is a subfield of Natural Language Processing (NLP). In this project, we train a neural network to learn the underlying structure and pattern in a given text corpus (tv script in our case) and then generate a never seen before (but fake) tv script. There are many real-life applications of text generating models, such as:

  • Product reports describing the features of a new product
  • Financial reports
  • Personalized emails.
  • Quickly generating unique marketing content targeting a unique customer.

Our objective here is to build an RNN based Neural Network that’d consume a text corpus (TV/film script, book, etc.), learn from it, and then given a seed `prime word; it’d generate a fake text snippet that’d look like if this generated text came from the original script/book.

  • We use a subset of the original Seinfeld dataset as our text corpus to train the model.
  • Unlike images or tabular data, the text data is sequential data where the order of input is crucial. Therefore, special preprocessing is required before it can be used to train the model.
    • Text data needs to be encoded into a numerical representation.
    • To train the model in the supervised setting, we’d need to split the input data between independent features (sequence of words) and the dependent/target feature (next word in the sequence). Data is then needed to be batched for model training.
  • Note that, unlike other supervised learning tasks, we can’t measure the performance of our text generating model using a metric like accuracy. Performance measurement, in this case, would be Qualitative (human evaluation) rather than Quantitative.
  • Since we deal with sequential data here, we’ll use an RNN based neural network best suited for such a task. RNN is well suited for sequential data because the order in which the data element appears in sequences is critical. RNN cell keeps track of the cell state (hidden state) of previous time-step along with input from current time-step to produce an output (which is again input to the next time-step as the previous hidden state). A typical high-level structure of the RNN cell is shown below…
  • In practice, the plain vanilla RNN shown above is hardly used nowadays, where equal importance is given to both the previous-hidden-state and the current input. Instead, the improved variants of RNN such as LSTM (Long Short Term Memory) or GRU (Gated Recurrent Unit) are used to tackle these issues. This project will use the GRU cell that is slightly lighter than the LSTM cell and trains a bit faster. If LSTM is used, we can get better results than what we are producing with GRU, but it might take longer to train. A typical GRU cell structure is shown below…
  • The neural network is defined as a subclass of the nn.Module Pytorch class. The network architecture is shown below…

  • The first layer of the network is an Embedding layer which takes an input vector of size equal to our vocabulary (number of unique words in our training text corpus) and outputs a vector representation of each of the words in the vocabulary.

  • The next layer is a GRU RNN cell that consumes word vectors from the embedding layer as input and produces hidden-state as outputs. Here we are using two layers of GRU cell, one stacked on top of the other hence n_layers parameter is set to 2.

  • To help model generalize well, 50% dropout for GRU cell and 50% dropout between GRU cell and the final fully-connected layer

  • We use the Adam optimizer with 0.0007 learning rate and CrossEntropyLoss as the loss function for the network training. Based on numerous experimentations, the final learning rate and hyperparameters listed above were selected…

    Hyperparameter Value
    sequence_length 15
    batch_size 64
    num_epoc 10
    learning_rate 0.0007
    embedding_dim 300
    hidden_dim 512
    n_layers 2

  • The model is trained with the above hyperparameters for ten epochs and achieved a loss of ~2.5

  • Note that the above generated fake script snippet appears to have come from the original TV script in terms of language, style, and tone; however, it’s far from perfect. Multiple characters say (somewhat) complete sentences. But, all sentences do not make sense. In practice, it takes quite a while, lot of experimentation and resources to get good results for such a complex task. We can use smaller vocabulary (and discard uncommon words) or get more data to improve outcomes. It’s still impressive to see how we could get a very trivial network to mimic a human-generated text up to some degree of accuracy with little effort; that’s a good start!
